## дёӯж–Ү

зәҜжң¬ең°гҖҒйӣ¶зҪ‘з»ңзҡ„ Android GPS йҒҘжөӢзӣ‘жҺ§еә”з”ЁгҖӮе°ҶжүӢжңәеҢ–иә«дёәиҲӘз©әзҺ»з’ғеә§иҲұд»ӘиЎЁзӣҳ вҖ” е®һж—¶ GNSS ж•°жҚ®гҖҒеӨҡжҳҹеә§еҚ«жҳҹиҝҪиёӘгҖҒйЈһиЎҢжЁЎејҸжЈҖжөӢпјҢе…ЁйғЁд»ҘиҲӘз©әзә§ж·ұиүІдё»йўҳе‘ҲзҺ°гҖӮ

### еҠҹиғҪ

#### Dashboard д»ӘиЎЁзӣҳ
- **GNSS еқҗж Ү** вҖ” з»Ҹзә¬еәҰ DD/DMS еҸҢж јејҸпјҢжҜҸз§’еҲ·ж–°
- **зҰ»зәҝең°еӣҫ** вҖ” osmdroid з“ҰзүҮзј“еӯҳпјҢд»…жҳҫзӨәдҪҚзҪ®е…үзӮ№пјҢж— еҹҺеёӮж ҮзӯҫпјҢж— йңҖзҪ‘з»ң
- **жҳҹеә§жҰӮи§Ҳ** вҖ” жҜҸдёӘжҳҹеә§зҡ„еҸҜи§ҒеҚ«жҳҹж•°гҖҒеҸӮдёҺе®ҡдҪҚж•°гҖҒжңҖдҪі SNR
- **ж°”еҺӢй«ҳеәҰ**е’Ң**ең°йқўйҖҹеәҰ**пјҲkm/hпјү
- **SNR жҹұзҠ¶еӣҫ** вҖ” жҢүжҳҹеә§зқҖиүІпјҢжҢүдҝЎеҸ·ејәеәҰжҺ’еәҸ
- **NMEA ж—Ҙеҝ—жөҒ** вҖ” е®һж—¶ж»ҡеҠЁ GPS еҺҹе§ӢиҜӯеҸҘ

#### Skyview еҚ«жҳҹеӨ©еӣҫ
- **йӣ·иҫҫжү«жҸҸеҷЁ** вҖ” еҠЁз”»жү«жҸҸпјҢжҳҹеә§зқҖиүІеҚ«жҳҹе…үзӮ№жҢүд»°и§’/ж–№дҪҚи§’з»ҳеҲ¶
- **жҳҹеә§еӣҫдҫӢ** вҖ” еҸҜеҲҮжҚўеҗ„зі»з»ҹжҳҫзӨә/йҡҗи—Ҹ
- **еҚ«жҳҹжё…еҚ•иЎЁ** вҖ” SVIDгҖҒжҳҹеә§гҖҒд»°и§’гҖҒж–№дҪҚи§’гҖҒSNR (dB-Hz)гҖҒй”Ғе®ҡзҠ¶жҖҒпјҲйўңиүІж Үи®°пјү
- **еҒҘеә·ж‘ҳиҰҒ** вҖ” жҖ»еҸҜи§Ғж•°гҖҒеҸӮдёҺе®ҡдҪҚж•°гҖҒе…Ёзі»з»ҹжңҖдҪі SNR

#### Trends и¶ӢеҠҝ
- **йҖҹеәҰиЎЁ** вҖ” зҺҜеҪўиҝӣеәҰжҢҮзӨәеҷЁпјҢе®һж—¶ km/h иҜ»ж•°
- **й«ҳеәҰи¶ӢеҠҝ** вҖ” иҝ‘жңҹй«ҳеәҰеҺҶеҸІжҠҳзәҝеӣҫ
- **еһӮзӣҙйҖҹеәҰжҢҮзӨәеҷЁ (VSI)** вҖ” зҲ¬еҚҮ/дёӢйҷҚйҖҹзҺҮ m/s
- **ең°еҪўиғҢжҷҜ** вҖ” еӣҫиЎЁеҗҺж–№еҫ®еҰҷзҡ„жө·жӢ”жёҗеҸҳ

#### Settings и®ҫзҪ®
- **еҚ•дҪҚеҲҮжҚў** вҖ” е…¬еҲ¶ (km/h, m) / иӢұеҲ¶ (mph, ft)
- **еқҗж Үж јејҸ** вҖ” DDпјҲеҚҒиҝӣеҲ¶еәҰпјүгҖҒDMSпјҲеәҰеҲҶз§’пјүгҖҒUTM
- **й«ҳеәҰеҹәеҮҶ** вҖ” WGS84 жӨӯзҗғйқўжҲ– MSL ж ЎжӯЈ
- **зҰ»зәҝж•°жҚ®** вҖ” еҜјеҮә NMEA ж—Ҙеҝ—гҖҒжё…йҷӨзј“еӯҳең°еӣҫз“ҰзүҮ
- **йҡҗз§ҒеЈ°жҳҺ** вҖ” жүҖжңүж•°жҚ®з•ҷеңЁи®ҫеӨҮдёҠпјҢйӣ¶зҪ‘з»ңдј иҫ“

#### йЈһиЎҢжЁЎејҸжЈҖжөӢ
иҮӘеҠЁиҜҶеҲ«й«ҳйҖҹ/й«ҳз©әзҺҜеўғпјҡ
- **йҖҹеәҰ > 200 km/h** вҶ’ `FLIGHT?` жҢҮзӨәеҷЁ
- **й«ҳеәҰ > 8,000 m** вҶ’ `HIGH ALT` жҢҮзӨәеҷЁ

Y иҪҙйҮҮз”ЁеўЁеҚЎжүҳжҠ•еҪұжқҘзә жӯЈзӯүи·қзҹ©еҪўжҠ•еҪұеңЁжһҒең°йҷ„иҝ‘зҡ„з•ёеҸҳгҖӮзә¬еәҰиў«й’іеҲ¶еңЁ Вұ85В° д»ҘйҒҝе…ҚжһҒзӮ№зҡ„еҘҮејӮжҖ§пјҢз„¶еҗҺйҖҡиҝҮйҖҶ Gudermannian еҮҪж•°жҳ е°„пјҡ`yMerc = ln(tan(ПҖ/4 + ПҶ/2))`гҖӮз»“жһңеҪ’дёҖеҢ–еҲ° 0вҖ“60 зҡ„зҪ‘ж јиҢғеӣҙпјҢе‘ҲзҺ°еҮәи§Ҷи§үдёҠзҶҹжӮүзҡ„дё–з•Ңең°еӣҫ вҖ” ж јйҷөе…°е’ҢеҚ—жһҒжҙІзҡ„жҜ”дҫӢеӨ§е°Ҹз¬ҰеҗҲзӣҙи§үгҖӮ

#### еӣәе®ҡе®Ҫй«ҳжҜ”дёҺ Letterbox

ж— и®әеҸҜз”Ё Canvas з©әй—ҙеҰӮдҪ•еҸҳеҢ–пјҢең°еӣҫе§Ӣз»ҲдҝқжҢҒ 126:60 (2.1:1) зҡ„еӣәе®ҡе®Ҫй«ҳжҜ”пјҡ

```
иӢҘ canvasAspect > worldAspect (е®ҪеұҸ)пјҡ
  вҶ’ жҢүй«ҳеәҰйҖӮй…ҚпјҢж°ҙе№і letterbox
  вҶ’ mapH = canvasH Г— scale, mapW = mapH Г— 2.1

иӢҘ canvasAspect < worldAspect (з«–еұҸ)пјҡ
  вҶ’ жҢүе®ҪеәҰйҖӮй…ҚпјҢеһӮзӣҙ letterbox
  вҶ’ mapW = canvasW Г— scale, mapH = mapW / 2.1

зҪ‘ж јеҚ•е…ғе°әеҜёпјҡ
  cellW = mapW / 126, cellH = mapH / 60
  з”ұдәҺ mapW/mapH = 126/60 вҶ’ cellW == cellH (жӯЈж–№еҪўеҚ•е…ғ)
```

Letterbox еҢәеҹҹдҪңдёәеӨ©з„¶зҡ„и§ҶеҸЈиҫ№з•Ң вҖ” ең°еӣҫеҶ…е®№е®ҢзҫҺеұ…дёӯпјҢеӨҡдҪҷз©әй—ҙд»ҘиғҢжҷҜиүІеЎ«е……гҖӮиҝҷжҳҜи§ЈеҶіж—©жңҹзүҲжң¬"ең°еӣҫеҸҳеҪў"й—®йўҳзҡ„ж №жң¬ж–№жЎҲгҖӮ

#### GPS е®ҡдҪҚж Үи®°

еҪ“иҺ·еҸ–еҲ° GNSS е®ҡдҪҚж—¶пјҢз”ЁжҲ·дҪҚзҪ®д»ҘйҶ’зӣ®зҡ„еҸ‘е…үеҚҒеӯ—еҮҶжҳҹз»ҳеҲ¶еңЁең°еӣҫдёҠпјҡ

- **еӨ–еұӮи„үеҶІзҺҜ** вҖ” 12px еҚҠеҫ„пјҢPrimary иүІ 35% йҖҸжҳҺеәҰ
- **дёӯеұӮе®һеҝғзҺҜ** вҖ” 5px е®һеҝғ Primary
- **ж ёеҝғеңҶзӮ№** вҖ” 2.5px PrimaryFixedпјҲдёҚйҖҸжҳҺпјү
- **еҚҒеӯ—еҮҶжҳҹ** вҖ” еӣӣдёӘж–№еҗ‘еҗ„ 10pxпјҢ60% йҖҸжҳҺ
- **еқҗж Үж Үзӯҫ** вҖ” `lat, lon` зӯүе®Ҫеӯ—дҪ“пјҢиҮӘйҖӮеә”еӨ§е°Ҹ (7вҖ“12sp)

дҪҚзҪ®д»Һ WGS84 еқҗж ҮйҖҡиҝҮеўЁеҚЎжүҳзҪ‘ж јеҸҳжҚўжҳ е°„дёә Canvas еғҸзҙ еқҗж ҮпјҢдёҺжүҖжңүең°еӣҫе…ғзҙ е…ұз”ЁеҗҢдёҖеҘ—жҠ•еҪұз®—жі•гҖӮ

- **е°ҡжңӘе®ҡдҪҚ** вҶ’ еӣһйҖҖеұ…дёӯдәҺдёӯеӣҪдёӯеҝғ (35В°N, 105В°E)
- `centeredOnFix` ж Үеҝ—зЎ®дҝқжҜҸж¬Ўе®ҡдҪҚиҺ·еҸ–д»…жү§иЎҢдёҖж¬Ўеұ…дёӯ вҖ” еҗҺз»ӯжӢ–еҠЁе®Ңе…Ёз”ұз”ЁжҲ·жҺ§еҲ¶

- **иө·зӮ№ж Үи®°** вҖ” 4px е®һеҝғеңҶпјҢ80% йҖҸжҳҺ
- **з»ҲзӮ№ж Үи®°** вҖ” 4px е®һеҝғеңҶпјҢPrimaryFixedпјҲдёҚйҖҸжҳҺе®үе…Ёй»„пјү

- `DotMatrixMap(isFullscreen = true)` жёІжҹ“зәҜ Canvas е…ЁеұҸең°еӣҫ
- йҖҡиҝҮиҝ”еӣһжүӢеҠҝжҲ–зі»з»ҹиҝ”еӣһй”®йҖҖеҮәпјҢжҒўеӨҚз«–еұҸ

1. **дё»йўҳеҚід»Јз Ғ** вҖ” Astra Precision и®ҫи®Ўд»ӨзүҢ 1:1 з§»жӨҚеҲ° `Color.kt`/`Type.kt`/`Shape.kt`пјҢеҗҺз»ӯж–°еўһйЎөйқўиҮӘеҠЁз»§жүҝиҲӘз©әзҫҺеӯҰгҖӮ

8. **зӮ№йҳөең°еӣҫжүӢеҠҝжј”иҝӣ** вҖ” ең°еӣҫжүӢеҠҝз»ҸеҺҶдәҶеӣӣиҪ®иҝӯд»Јпјҡv1.4.2 е°Ҷең°еӣҫд»Һ ScrollView дёӯз§»еҮәпјҲи§ЈеҶідәӢд»¶иў«зҲ¶з»„д»¶жӢҰжҲӘпјүпјӣv1.6.1 е°қиҜ•жүӢеҠЁ `awaitPointerEventScope` еә•еұӮдәӢд»¶еӨ„зҗҶпјҲеӣ  `awaitEachGesture` ж—  Compose 1.5.x ж”ҜжҢҒиҖҢж”ҫејғпјүпјӣv1.6.2 е®һзҺ° anchored zoom дҪҶзјә `mapLeft` иЎҘеҒҝпјҲж”ҫеӨ§ж—¶еҗ‘е·ҰдёҠи§’жјӮз§»пјүпјӣv1.6.4 еӣһеҪ’ `detectTransformGestures` е№¶иЎҘе…Ё letterbox еҒҸз§»иЎҘеҒҝпјҢжңҖз»Ҳж №жІ»жүҖжңүдәӨдә’й—®йўҳгҖӮ

## English

Pure-local, network-free Android app that turns your device into a glass-cockpit instrument cluster. Real-time GNSS data, multi-constellation satellite tracking, and flight mode detection вҖ” all rendered in an aviation-grade dark theme.

### Features

#### Dashboard
- **GNSS coordinates** вҖ” latitude/longitude in DD and DMS formats, refreshed every second
- **Offline map** вҖ” osmdroid tile cache shows current position as a glowing dot; no city labels, no network needed
- **Constellation summary** вҖ” per-constellation satellite count, used-in-fix count, best SNR
- **Barometric altitude** and **ground speed** in km/h
- **SNR bar graph** вҖ” colour-coded by constellation, sorted by signal strength
- **NMEA log stream** вҖ” live scrolling raw sentences from the GPS receiver

#### Skyview
- **Radar scanner** вҖ” animated sweep with constellation-coloured satellite dots plotted by elevation/azimuth
- **Constellation legend** вҖ” toggle individual systems on/off
- **Satellite table** вҖ” SVID, constellation, EL/AZ, SNR (dB-Hz), lock status with colour coding
- **Health summary** вҖ” total visible, used in fix, best SNR across all systems

#### Trends
- **Speed gauge** вҖ” circular arc progress indicator with real-time km/h readout
- **Altitude trend** вҖ” line chart of recent elevation history
- **Vertical speed indicator (VSI)** вҖ” climb/descent rate in m/s
- **Terrain background** вҖ” subtle elevation gradient behind charts

#### Settings
- **Units** вҖ” toggle between metric (km/h, m) and imperial (mph, ft)
- **Coordinate format** вҖ” DD (decimal degrees), DMS (degrees-minutes-seconds), UTM
- **Altitude reference** вҖ” WGS84 ellipsoid or MSL correction
- **Offline data** вҖ” export NMEA logs, clear cached map tiles
- **Privacy** вҖ” all data stays on device, zero network transmission

#### Flight Mode Detection
Automatically detects high-speed/high-altitude environments:
- **Speed > 200 km/h** вҶ’ `FLIGHT?` indicator
- **Altitude > 8,000 m** вҶ’ `HIGH ALT` indicator

#### Coordinate System

| Property | Value |
|:---|:---|
| Grid dimensions | 126 columns Г— 60 rows |
| Projection | Equirectangular (Plate CarrГ©e) |
| X-axis | -180В° (col 0) вҶ’ +180В° (col 125), linear |
| Y-axis | Mercator transform: `y = 60 Г— (1 - ln(tan(ПҖ/4 + ПҶ/2)) / mercMax) / 2` |
| Cell shape | Square: `cellW == cellH` guaranteed by fixed aspect ratio |
| Canvas fit | 126:60 letterbox вҖ” pad shorter axis, never stretch |

The Y-axis uses a Mercator projection to correct polar distortion inherent in equirectangular maps. Latitudes are clamped to Вұ85В° to prevent the singularities at the poles, then mapped through the inverse Gudermannian function: `yMerc = ln(tan(ПҖ/4 + ПҶ/2))`. The result is normalized to the 0вҖ“60 grid range, producing a visually familiar world map where Greenland and Antarctica appear proportionally sized.

#### Fixed Aspect Ratio & Letterbox

The map enforces a strict 126:60 (2.1:1) aspect ratio regardless of the available canvas space:

```
If canvasAspect > worldAspect (wide screen):
  вҶ’ Fit to canvas height, letterbox horizontally
  вҶ’ mapH = canvasH Г— scale, mapW = mapH Г— 2.1

If canvasAspect < worldAspect (tall screen):
  вҶ’ Fit to canvas width, letterbox vertically
  вҶ’ mapW = canvasW Г— scale, mapH = mapW / 2.1

Grid cell size:
  cellW = mapW / 126, cellH = mapH / 60
  Since mapW/mapH = 126/60 вҶ’ cellW == cellH (square cells)
```

The letterbox region acts as a natural viewport вҖ” the map content is perfectly centered, with extra space rendered as the background color. This is the root fix for the "map distortion" issue that plagued earlier versions.

#### GPS Position Marker

When a GNSS fix is acquired, the user's position is drawn on the map with a distinctive glowing crosshair:

- **Outer pulse ring** вҖ” 12px radius, Primary colour at 35% alpha
- **Mid ring** вҖ” 5px solid Primary
- **Core dot** вҖ” 2.5px PrimaryFixed (opaque)
- **Crosshair** вҖ” 20px lines in cardinal directions, 60% alpha
- **Coordinate label** вҖ” `lat, lon` in monospace font, scale-adaptive size (7вҖ“12sp)

Position is mapped from WGS84 coordinates to canvas pixel coordinates through the same Mercator grid transform used for all map elements.

### Development Notes

1. **Theme as Code** вҖ” Astra Precision design tokens ported 1:1 into `Color.kt`, `Type.kt`, `Shape.kt`. Every new screen automatically inherits the aviation aesthetic.
2. **Font Hosting** вҖ” JetBrains Mono + Inter bundled as Android `font` resources; works offline from first launch (unlike the HTML prototype's Google Fonts CDN).
3. **osmdroid over Google Maps** вҖ” No Play Services dependency, configurable disk cache, labels disabled for pure position display.
4. **GnssStatus.Callback over GpsStatus.Listener** вҖ” Modern API (24+) enables `getConstellationType()` for 8-system differentiation, plus `hasAlmanacData()`/`hasEphemerisData()` per satellite.
5. **Single ViewModel** вҖ” Four screens share one GPS data source; lifecycle managed in `MainActivity` not per-screen. Tab switching doesn't restart the GNSS engine.
6. **Speed Noise** вҖ” `Location.getSpeed()` returns m/s with low-velocity jitter. Raw readings displayed; future improvement: exponential moving average filter.
7. **Build on Bare JDK** вҖ” No Android Studio required. Key: `local.properties` вҶ’ SDK root, `ANDROID_HOME` env, `gradle.properties` вҶ’ `android.useAndroidX=true`.
8. **Dot-Matrix Map Gesture Evolution** вҖ” The map gesture system evolved through 4 iterations: v1.4.2 removed the map from a ScrollView (to stop the parent intercepting touch events); v1.6.1 attempted manual `awaitPointerEventScope` low-level handling (abandoned вҖ” `awaitEachGesture` unavailable in Compose 1.5.x); v1.6.2 implemented anchored zoom but missed `mapLeft` compensation (causing top-left drift); v1.6.4 returned to `detectTransformGestures` and added full letterbox offset compensation, resolving all interaction bugs.
9. **Negative Modulo Gotcha** вҖ” Kotlin's `%` operator returns negative values for negative operands (unlike Python/math). This caused the map grid to shift by exactly 48 columns in v1.6.3. Removing the modulo entirely and using raw offset values was the simplest fix.

**Key Design Principles**:
- **Single ViewModel**: All four screens share `GpsViewModel`, lifecycle bound to `MainActivity`. Tab switches don't restart GNSS.

#### 2. Red-Line Code (Do NOT Modify Without Full Context)

The following code took multiple debug iterations to stabilize. **Understand the history before touching**:

| File / Location | Code | Why It Must Not Change |
|:---|:---|:---|
| `DotMatrixMap.kt` anchored zoom | `offsetX = centroid.x - newLeft - actualZoom * (centroid.x - oldLeft - offsetX)` | Missing `newLeft`/`oldLeft` compensation causes top-left drift during zoom (v1.6.2 bug). `mapLeft` changes with scale; must use old/new pairs. |
| `DotMatrixMap.kt` `gridStartX` | `val gridStartX = mapLeft + offsetX` (**no modulo**) | v1.6.3 used `% cellW` causing negative-modulo bug, shifting grid 48 columns. Kotlin `%` returns negative for negative operands, unlike Python. **Never add modulo back**. |
| `DotMatrixMap.kt` `latLngToCanvas` | `gridStartX + gx * cellW` (**no +cellW/2**) | v1.7.1 removed the extra half-cell offset. `gx` is a continuous coordinate; adding half a cell misaligns city labels with land dots, especially in fullscreen. |
| `DotMatrixMap.kt` `centeredOnFix` | Centers once on first fix, then user-controlled | If offset resets on every fix, user pans get instantly overridden. `centeredOnFix` ensures "center exactly once" semantics. |
| `DotMatrixMap.kt` fixed aspect | `worldAspect = 126f / 60f` + letterbox | If stretched to actual canvas dimensions in landscape fullscreen, the map distorts. Fixed 2.1:1 + letterbox is the only correct approach. |
| `GpsRepository.kt` satellite dedup | `satMap[constellationType to svid] = info` | Some devices report the same satellite as UNKNOWN + real constellation. Without dedup, LazyColumn key collision crashes (v1.6.4 crash root cause). |
| `SkyviewScreen.kt` LazyColumn key | `key = { "${it.constellation.constellationType}_${it.svid}" }` | Must use `constellationType` (Int), not `label` (String). UNKNOWN constellation has cross-system svid overlaps. **Never use label + svid**. |
| `GpsViewModel.kt` network monitor | `registerDefaultNetworkCallback` + `checkNetworkImmediate()` | v1.6.2 used `registerNetworkCallback(NetworkRequest)` вҖ” callbacks didn't fire on some devices. `registerDefaultNetworkCallback` is more reliable; pair with active query to avoid races. |
| `GpsRepository.kt` NaN sanitization | `status.getCn0DbHz(i).let { if (it.isNaN() \|\| it < 0) 0f else it }` | `getCn0DbHz` returns NaN when a satellite has no signal. `coerceIn` is a no-op on NaN, causing SNR bar and avgSnr calculation errors. |
| `WorldMapGrid.kt` RLE bitmap | `bitmap: ByteArray` + `isLand(x, y)` | This is a pre-generated 126Г—60 land bitmap. **Do not hand-edit bytes**. Regenerate via script if coastlines need updating. |
| `FullscreenMapScreen.kt` landscape | `requestedOrientation = SCREEN_ORIENTATION_SENSOR_LANDSCAPE` | Fullscreen map must be landscape, otherwise dot layout scrambles. `DisposableEffect` restores `UNSPECIFIED` on exit. |

#### 3. Common Issues & Troubleshooting

**1. Map won't drag / zoom drifts**
- Check if `DotMatrixMap` is nested in `ScrollView`/`Column` without `.clipToBounds()` вҖ” parent intercepts touch events
- Check if `detectTransformGestures` was replaced with manual `awaitPointerEventScope` вҖ” the latter is extremely error-prone in Compose 1.5.x, **always use the native API**
- Check if anchored zoom formula includes `oldLeft`/`newLeft` вҖ” omission causes top-left drift

**2. Skyview crashes on entry**
- 99% of the time, LazyColumn keys are not unique. Check `GpsRepository.updateSatelliteData()` for `(constellationType, svid)` dedup
- Check `SkyviewScreen` `items(key = ...)` uses `constellationType` (Int), not `label` (String)
- Check for NaN SNR вҖ” `coerceIn` is a no-op on NaN; must check `isNaN()` first

**3. Online mode shows "waiting for network" after toggle**
- `registerDefaultNetworkCallback` must be paired with `checkNetworkImmediate()` active query
- `setOnlineMode()` must call `repository.restart()` to restart GPS monitoring
- Call `checkNetworkImmediate()` immediately on toggle вҖ” don't wait for callback

**4. Map distortion / city label misalignment**
- Check if `gridStartX` has `% cellW` added вҖ” remove it
- Check if `latLngToCanvas` has `+ cellW/2` added вҖ” remove it
- Check `mapW`/`mapH` follows fixed 126:60 aspect ratio вҖ” don't stretch to actual canvas dims

**5. Build failure: Java / SDK not found**
- `JAVA_HOME` must point to JDK 17 (path: `C:/Users/Fox/.workbuddy/android-tools/jdk-17.0.19+10`)
- `local.properties` must have `sdk.dir` pointing to Android SDK root
- Gradle 8.2 + Kotlin 1.9.21 + Compose BOM 2023.10.01 is the verified combo вҖ” **do not upgrade casually**

**6. GitHub Release Chinese text garbled**
- PowerShell heredoc with curl `--data-binary` corrupts UTF-8 encoding
- Fix: write JSON to a standalone `.json` file, use `curl -d @file.json`
